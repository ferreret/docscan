# Progreso 2026-03-20

## Resumen

Sesión de refactorización profunda, corrección de bugs y pruebas manuales. Se eliminó el paso de pipeline AiStep (la funcionalidad IA se programa en ScriptStep con los SDKs), se unificaron los campos de página en un solo `page.fields`, se corrigieron memory leaks en ventanas y se completaron las pruebas manuales de la release 1B.

## Cambios realizados

### 1. Eliminación de AiStep y renombrado ai_fields (commits d8604ab, d3bfe21, 7f4ba3b)

- **AiStep eliminado** del pipeline: `steps.py`, `executor.py`, `serializer.py`, `tab_pipeline.py`
- Borrados: `app/services/ai_service.py`, `app/ui/configurator/step_dialogs/ai_step_dialog.py`
- `ai_fields` renombrado a `custom_fields` y luego **unificado con `page.fields`** — ahora hay un solo dict por página
- Columna `custom_fields_json` eliminada de la BD (migrada a `index_fields_json`)
- `PageState.AI_FIELDS` → `PageState.HAS_FIELDS`
- Constante `REMOVED_STEP_TYPES` en `steps.py` para compatibilidad con pipelines antiguos
- El serializer tolera pasos `"ai"` en JSON antiguo (log debug + skip)
- **2 migraciones Alembic** creadas y aplicadas:
  - `c7e1f3a0b521`: renombrar `ai_fields_json` → `custom_fields_json` + limpiar pasos AI del pipeline_json
  - `d9a2b4c1e835`: merge `custom_fields_json` en `index_fields_json` + drop columna

### 2. Corrección de memory leak en ventanas (commit ee7c378)

- **WorkbenchWindow**: añadido `WA_DeleteOnClose`, `deleteLater()`, limpieza de referencias pesadas en `closeEvent`
- **BatchManagerWindow**: mismo fix
- Guards en `_on_page_processed`, `_on_all_processed`, `_fire_event` para ignorar señales tardías tras cierre
- **Reanudación automática del pipeline**: al reabrir un lote, las páginas no procesadas se detectan y encolan automáticamente

### 3. Navegación next-barcode/next-review + fix events_json (commit e8554e6)

- **ViewerOverlay**: nuevos botones y señales `nav_next_barcode`, `nav_next_review`
- **WorkbenchWindow**: métodos `_on_next_barcode` (siguiente página con barcodes) y `_on_next_review` (siguiente pendiente de revisión)
- **worker_main.py**: corregido parseo de `events_json` para aceptar ambos formatos (str directo y dict con script/entry_point)
- **CLAUDE.md**: eliminados `ConditionStep` y `HttpRequestStep` del pipeline (ScriptStep los cubre)

### 4. Mejoras UI (commit 96e7048)

- Configurador: tamaño inicial 950×700 (antes 800×600 mínimo)
- Workbench: tamaño inicial 1440×900 (antes 1280×800 mínimo)
